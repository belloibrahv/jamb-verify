# Troubleshooting Payment and Verification Issues

## Critical Fixes Applied

### Issue 1: Verify Endpoint Returning 500 Errors

**Root Cause**: The verify endpoint was failing because:
1. Transaction record lookup was failing silently
2. Wallet update was not checking if rows were affected
3. Error messages were not detailed enough for debugging

**Solution Applied**:
- Added comprehensive logging with `[VERIFY]` prefix for easy filtering
- Added `.returning()` to wallet update to verify the operation succeeded
- Improved error messages to include specific failure reasons
- Increased retry attempts from 2 to 3
- Added idempotency check to prevent double-processing

### Issue 2: Balance Not Updating After Payment

**Root Cause**: Multiple factors:
1. Verify endpoint was failing (see Issue 1)
2. Client-side error handling was not capturing the actual error
3. Balance comparison logic was checking against 0 instead of previous balance

**Solution Applied**:
- Fixed verify endpoint (see Issue 1)
- Added detailed client-side logging with `[PAYMENT]` prefix
- Capture and display actual error messages from verify endpoint
- Show reference number in error messages for support
- Display final balance in success message

### Issue 3: Webhook Not Processing Correctly

**Root Cause**:
- Silent failures in database operations
- No verification that wallet record exists before update
- Insufficient logging for debugging

**Solution Applied**:
- Added comprehensive logging with `[WEBHOOK]` prefix
- Added `.returning()` to verify wallet update succeeded
- Check if wallet exists and throw specific error if not
- Log all retry attempts with delays

## How to Debug Payment Issues

### Step 1: Check Browser Console

Open DevTools (F12) and look for logs with these prefixes:
- `[PAYMENT]` - Client-side payment flow
- Look for the payment reference (e.g., `jv_abc123`)

**Success Pattern**:
```
[PAYMENT] Starting verification for reference: jv_abc123
[PAYMENT] Verify attempt 1/3 for reference: jv_abc123
[PAYMENT] Verify response (attempt 1): {status: 200, ok: true, data: {...}}
[PAYMENT] Verification successful
[PAYMENT] Balance fetch attempt 1/5
[PAYMENT] Balance fetched (attempt 1): {current: 50000, previous: 0, increased: true}
[PAYMENT] Balance increased! Update successful.
[PAYMENT] Payment flow completed successfully!
```

**Failure Pattern - Verify Fails**:
```
[PAYMENT] Verify attempt 1/3 for reference: jv_abc123
[PAYMENT] Verify response (attempt 1): {status: 500, ok: false, data: {message: "..."}}
[PAYMENT] Verify failed (attempt 1): Failed to update wallet
[PAYMENT] Verification failed after 3 attempts
```

### Step 3: Common Error Patterns and Solutions

#### Error: "Transaction record not found for reference: jv_xxx"

**Meaning**: The payment was made but no transaction record exists in the database.

**Possible Causes**:
1. Initialize endpoint failed to create the transaction record
2. Database connection issue during initialization
3. Payment was made with a reference not generated by our system

**Debug Steps**:
1. Check Vercel logs for `/api/paystack/initialize` with that reference
2. Look for database errors during transaction creation
3. Verify the reference format matches `jv_XXXXXXXXXX`

**Solution**:
- If initialize failed: Check database connection and retry payment
- If reference is wrong: This payment should be refunded via Paystack dashboard

#### Error: "Wallet not found for user: xxx"

**Meaning**: User exists but has no wallet record.

**Possible Causes**:
1. Wallet was not created during user registration
2. Wallet was deleted
3. Database migration issue

**Debug Steps**:
1. Check if user exists in database
2. Check if wallet exists for that user
3. Review user registration flow

**Solution**:
```sql
-- Manually create wallet for user
INSERT INTO wallets (id, user_id, balance, currency, updated_at)
VALUES ('wallet_' || gen_random_uuid(), 'USER_ID_HERE', 0, 'NGN', NOW());
```

#### Error: "Payment verification failed: Paystack API call failed"

**Meaning**: Cannot reach Paystack API or invalid credentials.

**Possible Causes**:
1. Invalid Paystack secret key
2. Network issue
3. Paystack API is down

**Debug Steps**:
1. Verify `PAYSTACK_SECRET_KEY` in Vercel environment variables
2. Test Paystack API directly:
```bash
curl https://api.paystack.co/transaction/verify/REFERENCE \
  -H "Authorization: Bearer YOUR_SECRET_KEY"
```

**Solution**:
- Update secret key if invalid
- Wait and retry if Paystack is down
- Check Paystack status page: https://status.paystack.com

#### Error: "Payment not successful. Status: pending"

**Meaning**: Payment was initiated but not completed.

**Possible Causes**:
1. User closed popup before completing payment
2. Card was declined
3. Payment is still processing

**Debug Steps**:
1. Check Paystack dashboard for transaction status
2. Ask user if they completed the payment
3. Wait a few minutes and check again

**Solution**:
- If truly pending: Wait or ask user to retry
- If failed: Transaction will remain pending, no refund needed
- If successful in Paystack but showing pending: Manually trigger webhook

#### Error: "Balance increased! Update successful" but UI shows old balance

**Meaning**: Backend updated successfully but frontend state is stale.

**Possible Causes**:
1. React state not updating
2. Cache issue
3. Multiple tabs open

**Solution**:
- Click "Refresh balance" button
- Hard refresh page (Cmd+Shift+R / Ctrl+Shift+R)
- Close other tabs and retry

### Step 4: Manual Verification and Recovery

If payment succeeded in Paystack but balance didn't update:

1. **Get the payment reference** from browser console or Paystack dashboard

2. **Manually trigger verify endpoint**:
```bash
curl "https://jamb-verify.vercel.app/api/paystack/verify?reference=jv_abc123" \
  -H "Cookie: your_session_cookie"
```

3. **Check the response**:
- If 200: Balance should update
- If 500: Check error message and Vercel logs
- If 400: Payment may not be successful in Paystack

4. **If verify fails, check Paystack dashboard**:
- Go to https://dashboard.paystack.com/#/transactions
- Search for the reference
- Verify status is "Success"
- Note the amount

5. **If Paystack shows success but verify still fails**:
- Check Vercel logs for the specific error
- Verify transaction record exists in database
- Verify wallet exists for user
- Contact support with reference number

## Testing Checklist

Before marking this as resolved, test the following:

### Test 1: Successful Payment Flow
1. Log in to dashboard
2. Note current balance
3. Click "Fund wallet" with ₦500
4. Complete payment with test card: `4111 1111 1111 1111`
5. Wait for popup to close
6. Verify:
   - Success message appears
   - Balance increases by ₦500
   - No errors in console

### Test 2: Cancelled Payment
1. Click "Fund wallet"
2. Close popup without paying
3. Verify:
   - "Payment cancelled" message appears
   - Balance unchanged
   - No errors in console

### Test 3: Failed Payment
1. Click "Fund wallet"
2. Use declined test card: `5060 6666 6666 6666 666`
3. Verify:
   - Error message appears
   - Balance unchanged
   - Transaction status is "failed" or "pending"

### Test 4: Network Issues
1. Open DevTools > Network tab
2. Set throttling to "Slow 3G"
3. Complete payment
4. Verify:
   - Retries happen automatically
   - Eventually succeeds or shows clear error
   - No infinite loops

### Test 5: Concurrent Payments
1. Open two browser tabs
2. Initiate payment in both tabs
3. Complete both payments
4. Verify:
   - Both payments process correctly
   - Balance increases by sum of both amounts
   - No duplicate processing

## NIN Verification Issues

The NIN verification is a separate issue from payment. If NIN verification fails:

### Check YouVerify API

1. **Verify endpoint is correct**:
```bash
# Should be v2, not v1
echo $YOUVERIFY_BASE_URL
# Expected: https://api.sandbox.youverify.co/v2
```

2. **Test API directly**:
```bash
curl -X POST https://api.sandbox.youverify.co/v2/identities/verifications/ng/nin \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "11111111111",
    "isSubjectConsent": true
  }'
```

3. **Check response**:
- 200: API is working
- 401: Invalid token
- 402: Insufficient balance (sandbox should not have this)
- 404: Wrong endpoint (check if using v1 instead of v2)

### Common NIN Verification Errors

#### Error: "We couldn't complete the verification"

**Possible Causes**:
1. Wrong API endpoint (v1 instead of v2)
2. Invalid token
3. Insufficient wallet balance
4. Invalid NIN format

**Solution**:
1. Verify `.env` has `YOUVERIFY_BASE_URL="https://api.sandbox.youverify.co/v2"`
2. Verify token is valid
3. Ensure wallet has at least ₦500 (50000 kobo)
4. Verify NIN is exactly 11 digits

## Support Process

When a user reports a payment issue:

1. **Get the payment reference** (format: `jv_XXXXXXXXXX`)
2. **Check Paystack dashboard** for transaction status
3. **Check Vercel logs** for that reference
4. **Check database** for transaction and wallet records
5. **Determine the issue** using patterns above
6. **Take action**:
   - If payment succeeded but balance not updated: Manually trigger verify
   - If payment failed: Explain to user, no action needed
   - If payment pending: Wait or ask user to retry
   - If duplicate: Refund via Paystack dashboard
